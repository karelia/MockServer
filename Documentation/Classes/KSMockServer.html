Overview

A server which runs locally and “pretends” to be something else.

You provide the server with an optional port to run on, and a list of responses.

The responses consist of an array of arrays, in this format: pattern, command, command…

The pattern is a regular expression which is matched against input received by the server. The commands are NSString, NSData, or NSNumber objects, which are processed when the pattern has been matched.

See the documentation.

See also the MockServerTests.m file for some examples.

runUntilStopped

Loop in the current run loop until something calls stop on the server.

- (void)runUntilStopped

Discussion

Typically you start the server, initiate the network operation that you want to test, then call runUntilStopped in your test case.

In a completion block or delegate method of your network operation, you can then call stop on the server, at which point your test case will continue executing and you can verify that you got the results you were expecting.
